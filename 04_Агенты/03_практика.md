# Практика — Модуль 04. Агенты и скиллы

## Задание 1. Определи свою боль

Прежде чем что-то создавать — пойми, зачем.

1. Запиши 5 задач, которые ты делаешь каждую неделю и которые бесят
2. Рядом с каждой напиши, сколько времени уходит
3. Выбери топ-1 — самую надоевшую

Пример:
```
1. Писать посты — 3 часа/неделю ← ВОТ ЭТА
2. Отвечать на типовые вопросы — 2 часа/неделю
3. Собирать отчёты — 1.5 часа/неделю
4. Искать файлы — 1 час/неделю
5. Составлять письма — 1 час/неделю
```

✅ **Готово, когда:** У тебя есть конкретная задача и понимание, сколько времени она жрёт.

## Задание 2. Создай первого агента по 7 шагам

Следуя инструкции (шаги 5-8), создай агента для задачи из Задания 1:

1. **Контекст** — убедись, что контекст проекта заполнен (Модуль 02)
2. **Должность** — заполни таблицу: работа, задачи, выход, качество
3. **Файл правил** — создай `.cursor/rules/[агент].mdc` со всеми секциями:
   - РОЛЬ
   - ЗАДАЧИ (основные + НЕ входит)
   - ПРАВИЛА
   - ИНСТРУКЦИИ
   - УТОЧНЯЮЩИЕ ВОПРОСЫ
   - САМООБУЧЕНИЕ
   - ПРИМЕРЫ
4. **Папка агента** — создай `агенты/[АГЕНТ]/` с подпапками `_КОНТЕКСТ/примеры/` и `_ЛОГИ/`
5. **Тестирование** — дай 3 задачи (простая, типичная, сложная)

✅ **Готово, когда:** Агент создан, папка структурирована, 3 тестовых задачи выполнены.

## Задание 3. Тройной прогон

1. Выбери важную задачу (текст для клиента, стратегия, важное письмо)
2. Выполни в Claude Sonnet
3. Переключи на GPT-4o → выполни ту же задачу
4. Сравни результаты
5. Запиши в `сравнение_моделей.md`:
   - Какая модель что сделала лучше
   - Какую выбрала бы для этой задачи
   - Почему

Бонус: попроси третью модель (Gemini) объединить лучшее из обоих результатов.

✅ **Готово, когда:** Файл `сравнение_моделей.md` создан с конкретными выводами.

## Задание 4. Система самообучения

1. Создай `lessons.md` в корне проекта
2. Дай агенту из Задания 2 минимум 5 разных задач
3. После каждой оцени: ✅ / ⚠️ / ❌
4. Для каждой ⚠️ или ❌ запиши в lessons.md:
   - Проблема (что не так)
   - Решение (что изменила в .mdc)
   - Результат (стало лучше?)
5. При оценке ✅ — сохрани запрос + результат в `_КОНТЕКСТ/примеры/`
6. Обнови `.mdc` файл на основе ошибок
7. Повтори одну из задач — стало лучше?

✅ **Готово, когда:** В lessons.md минимум 3 записи, в `_КОНТЕКСТ/примеры/` минимум 2 успешных кейса, повторный запрос лучше первого.

## Задание 5. Библиотека агентов

Создай минимум 2 дополнительных агента (итого 3+ вместе с первым).

Для каждого:
1. Определи должность (таблица из Задания 2)
2. Создай `.mdc` файл со всеми секциями
3. Создай папку `агенты/[АГЕНТ]/`
4. Протестируй на 2-3 задачах
5. Добавь секцию «связь с командой» — кому агент передаёт результат:
   ```
   ## СВЯЗЬ С КОМАНДОЙ
   После написания поста → передать @дизайнер для визуала
   ```

Идеи:
- Планировщик — разбивает задачи на шаги
- Ревьюер — проверяет тексты на ошибки
- Менеджер по продажам — составляет КП, отвечает на возражения
- Исследователь — ищет информацию, составляет обзоры

✅ **Готово, когда:** В `.cursor/rules/` лежит 3+ файлов .mdc, у каждого есть папка в `агенты/`, каждый протестирован.
