Где задолбался — туда агента

Боль = приоритет для агента

Не «что модно» или «что круто». А где реально горит.
  • Рутинные задачи — организация файлов, ответы на типовые вопросы
  • Контент — посты, статьи, описания товаров
  • Аналитика — отчёты, метрики, сбор данных
  • Коммуникации — письма, сообщения, координация
  • Документация — регламенты, инструкции, шаблоны
Примеры: Задолбался писать посты → Копирайтер Задолбался искать файлы → Уборщик документов Задолбался собирать отчёты → Аналитик

7 этапов создания агента

Структурированный процесс

  • 1 Контекст компании — что нужно знать агенту о вашем бизнесе?
  • 2 Определение должности — какую «работу» выполняет?
  • 3 Поиск кейсов — лучшие практики для этой роли
  • 4 Интервью с экспертом (опционально) — если есть реальный сотрудник
  • 5 Формирование плана — показываете, что будет создано
  • 6 Создание — файл правил + папка контекста + система самообучения
  • 7 Тестирование — даёте задачи → собираете обратную связь → агент учится

Что делает агент?

Вопросы для определения

  • Какую «работу» выполняет этот сотрудник?
  • Какие задачи делает ежедневно?

Принцип выбора из предыдущего модуля

Самый важный принцип

Система самообучения

Главное правило: Если запрос имеет логику → сохранить в контекст. Это отличает живую систему от статичного промпта.
  • 1 Сначала делаем (день 1) — базовый функционал, тестируем
  • 2 Докручиваем (день 2–7) — обратная связь, уроки, кейсы
  • 3 Масштабируем лучшее — копируем успешные паттерны
День 1 → получаем урок → День 2 → применяем урок → День 3 → используем кейс → результат лучше с первого раза!

  • После каждой задачи → запрос обратной связи

  • Если «Нужны правки» → записать урок в _ЛОГИ/
  • При следующей задаче → использовать опыт

3 типа тестовых задач

Как проверить агента

  • Простая Проверка базового функционала — «Привет» → агент отвечает
  • Типичная Обычная рабочая ситуация — «Организуй папку Downloads»
  • Сложная Проверка границ — «Организуй системную папку» → агент должен отказаться
Тестируйте на всех трёх типах задач. Это покажет границы возможностей агента.
АССИСТ+

Делаем → Докручиваем → Масштабируем

Процесс улучшения агента

  • 1 Сначала делаем — базовый агент, минимальный функционал, тестируем на 2–3 задачах
  • 2 Докручиваем — обратная связь после КАЖДОЙ задачи. За неделю: 10–20 задач, 5–10 уроков, 3–5 кейсов
  • 3 Масштабируем лучшее — при создании нового агента копируем успешные паттерны
Пример: Создали Копирайтера → накопили 20 кейсов → Создаёте SMM-менеджера → копируете лучшие кейсы → SMM-менеджер сразу пишет как Копирайтер после 20 итераций

Топ-5 ошибок при создании

Что не делать

Ошибки структуры

  • Нет папки агента (агенты/[АГЕНТ]/)
  • Неправильное имя файла (должно быть .mdc)
  • Нет папок _КОНТЕКСТ/примеры/ и _ЛОГИ/

Ошибки в правилах

  • Слишком длинные инструкции — агент теряется
  • Нет секции «НЕ входит» — агент берёт лишнее
  • Нет системы самообучения — агент не учится
Важно Ошибки использования: нет обратной связи после задач, игнорирование логов, ожидание идеального результата с первого раза.
