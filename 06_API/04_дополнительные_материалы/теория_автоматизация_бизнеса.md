# AI-автоматизация — теория и практика

## Зачем автоматизировать

Любой проект или бизнес тратит **30-40% времени** на повторяющиеся задачи:
- Ответы на однотипные вопросы клиентов
- Заполнение таблиц и отчётов
- Отправка писем и напоминаний
- Обработка заявок и лидов
- Публикация контента

AI-автоматизация убирает эту рутину. Не заменяет людей — освобождает их для важных задач.

---

## Подход этого курса: Cursor + API

В нашем модуле упор на **Cursor как основной инструмент**. Вы не пишете код руками — вы описываете задачу, Cursor пишет код.

### Что можно автоматизировать через Cursor
- Уведомления в Telegram
- Запись данных в Google Sheets
- Webhooks — приём входящих данных
- Связки между сервисами
- Работа с внешними сервисами через MCP

### Что сказать Cursor (примеры)
- «Напиши скрипт, который отправляет уведомление в Telegram когда приходит новая заявка»
- «Сделай бота, который записывает сообщения клиентов в Google Sheets»
- «Создай webhook-приёмник на Flask, который обрабатывает платежи от ЮKassa»

## Что автоматизировать в первую очередь

### Правило 80/20
Найди 20% задач, которые отнимают 80% времени. Обычно это:

1. **Обработка входящих** — заявки, письма, сообщения
2. **Отчётность** — сбор данных, формирование отчётов
3. **Коммуникации** — напоминания, follow-up, уведомления
4. **Контент** — посты, описания, рассылки

### Матрица приоритизации

| | Легко автоматизировать | Сложно автоматизировать |
|---|---|---|
| **Высокий эффект** | Делай ПЕРВЫМ | Планируй |
| **Низкий эффект** | Если есть время | Забудь |

## Связки (примеры)

- **Telegram Bot + Google Sheets** — приём заявок, учёт, уведомления
- **Webhook + Telegram** — мгновенные уведомления о событиях (оплата, регистрация)
- **Cursor + MCP + GitHub** — автоматизация задач в репозитории
- **Google Sheets + Telegram** — чтение данных из таблицы и отправка отчётов

Все эти связки вы делаете через Cursor — просто опишите задачу.

## MCP — главный тренд 2025-2026

**Model Context Protocol** — стандарт подключения AI к внешним сервисам.

Раньше: нужно было писать код интеграции для каждого сервиса.
Сейчас: устанавливаешь MCP-сервер и Cursor сам работает с сервисом.

### Примеры MCP
- **GitHub MCP** — работа с репозиториями, issues, PR
- **Filesystem MCP** — файлы за пределами проекта
- **Notion MCP** — управление базами данных
- **PostgreSQL MCP** — работа с БД
- **Playwright MCP** — автоматизация браузера

### Как подключить MCP к Cursor (3 способа)

**Способ 1: через настройки**
1. Cursor Settings → MCP
2. Add MCP Server
3. Выбери из списка или вставь команду

**Способ 2: через чат (самый простой)**
Напиши Cursor в чат:
```
Подключи MCP-сервер для GitHub. Мой токен: ghp_xxx
```
Cursor сам создаст конфигурацию.

**Способ 3: через файл конфигурации**
```json
// .cursor/mcp.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_..."
      }
    }
  }
}
```

## RAG и контекст

**Контекст** — файлы в папке проекта. Cursor читает их напрямую. Подходит для 10-50 файлов, не требует настройки.

**RAG** — документы индексируются в базу, AI сначала ищет нужный фрагмент, потом отвечает. Подходит для 1000+ документов, нужна настройка.

На практике: для большинства задач хватает контекста — просто положите файлы в папку.

## Типичные ошибки

1. **Автоматизировать всё сразу** — начни с одной задачи, доведи до ума, потом следующая
2. **Забыть про контроль** — AI ошибается. Ставь проверки на критичных этапах
3. **Не документировать** — через месяц забудешь, как работает. Пиши README
4. **Экономить на промптах** — плохой промпт = плохой результат. Инвестируй время в промпт
5. **Игнорировать безопасность** — не храни ключи в коде, используй .env файлы
