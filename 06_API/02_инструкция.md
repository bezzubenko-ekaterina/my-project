# Инструкция — Модуль 06. API и интеграции

## Шаг 1. Что такое API

API — это способ для программ общаться друг с другом. Когда ты пишешь боту в Telegram — твоё приложение отправляет запрос через Telegram API. Когда оплачиваешь в приложении — оно делает это через API платёжной системы.

Аналогия: API — это официант в ресторане. Ты (программа) говоришь официанту (API) что хочешь, он несёт заказ на кухню (сервер), получает блюдо (ответ) и приносит тебе.

Что нужно для работы с API:
- **URL** — адрес куда отправлять запрос
- **Ключ (API Key)** — твой пропуск (чтобы сервер знал кто ты)
- **Запрос (Request)** — что ты хочешь
- **Ответ (Response)** — что сервер вернул

Формат данных: почти все API используют **JSON** — текст с фигурными скобками:
```json
{
  "chat_id": "123456789",
  "text": "Привет!"
}
```

**Важно:** в этом модуле мы работаем через Cursor. Вам не нужно писать код руками — нужно понять логику и уметь правильно описать Cursor задачу.

Результат: Ты понимаешь что такое API и из чего состоит запрос.

## Шаг 2. Что такое API-ключ

API-ключ — это пароль от сервиса. Он доказывает, что запрос от тебя. Без ключа сервис не ответит. С чужим ключом — платит чужой человек.

Правила хранения ключей:
- Всегда в файле `.env` — **никогда** прямо в коде
- `.env` **обязательно** в `.gitignore`
- Если ключ попал на GitHub — немедленно отзови и создай новый
- Меняй ключи раз в 3-6 месяцев

**Что такое .env:** обычный текстовый файл в корне проекта, где хранятся секретные ключи. Каждая строка — одна настройка в формате `НАЗВАНИЕ=значение`.

**Что такое .gitignore:** файл, который говорит Git: «не загружай это на GitHub». Создай его ДО первого коммита.

Результат: Ты понимаешь зачем нужны ключи и как их правильно хранить.

## Шаг 4. Telegram Bot API — отправка уведомлений

Через Telegram API ваш бот может отправлять сообщения автоматически — уведомления о заявках, напоминания, алерты.

### Получи токен бота
Если у тебя есть бот из Модуля 05 — используй его токен. Если нет — создай нового в @BotFather.

### Узнай свой CHAT_ID
1. Напиши боту `@userinfobot` в Telegram
2. Он покажет твой ID (число вроде 775745451)
3. Добавь в `.env`: `TELEGRAM_CHAT_ID=775745451`

### Попроси Cursor написать скрипт

Что сказать Cursor:
```
Напиши скрипт на Python, который отправляет сообщение в Telegram через Bot API.
Токен бота (TELEGRAM_BOT_TOKEN) и CHAT_ID (TELEGRAM_CHAT_ID) бери из .env.
Используй requests.
Сообщение: "Привет! Это тестовое уведомление от моего скрипта."
```

Cursor напишет код. Запусти его и проверь Telegram — должно прийти сообщение от бота.

Результат: Ты умеешь отправлять сообщения в Telegram через Cursor.

## Шаг 5. Google Sheets API (один из множества примеров)

Google Sheets — это один из многих сервисов, который можно подключить через API. С его помощью можно автоматически записывать заявки, читать данные, обновлять статусы.

### Настройка (1 раз)
1. Зайди на console.cloud.google.com
2. Создай новый проект
3. В поиске найди «Google Sheets API» → Enable
4. Найди «Google Drive API» → Enable
5. IAM & Admin → Service Accounts → Create Service Account
6. Заполни имя → Create → Done
7. Кликни на созданный аккаунт → Keys → Add Key → Create new key → JSON
8. Скачается файл `credentials.json` — положи в папку проекта
9. Открой JSON — найди `"client_email": "...@...iam.gserviceaccount.com"`
10. Создай Google Таблицу → Поделиться → вставь этот email → Редактор

### Попроси Cursor написать скрипт

Что сказать Cursor:
```
Подключи Google Sheets API к проекту.
Напиши скрипт, который записывает строку в таблицу "Заявки".
Credentials бери из credentials.json.
Используй gspread.
Пример строки: ["Иван Петров", "+7 999 123-45-67", "Новая заявка", "2026-04-09"]
```

Результат: Ты умеешь работать с Google Sheets через Cursor.

## Шаг 6. Собери связку: Telegram + Google Sheets

Теперь объединим два сервиса в одну цепочку. Клиент пишет боту → данные записываются в таблицу → менеджер получает уведомление.

Что сказать Cursor:
```
Сделай Telegram-бота, который принимает заявки от клиентов.
Когда клиент пишет сообщение:
1. Бот извлекает имя и контакт из текста
2. Записывает данные в Google Sheets (имя, контакт, текст, дата)
3. Отправляет уведомление мне в Telegram

Токены и CHAT_ID бери из .env.
Credentials для Sheets — из credentials.json.
```

Результат: Работающая связка из двух сервисов.

## Шаг 7. Webhooks — сервис сам присылает данные

Webhook — это URL, на который сервис присылает данные когда что-то происходит. Вам не нужно постоянно спрашивать «есть новое?».

Примеры использования:
- **ЮKassa:** оплата прошла → webhook шлёт вам данные о платеже
- **GitHub:** новый коммит → уведомление в Telegram
- **Telegram:** бот получает сообщения мгновенно
- **Формы:** клиент заполнил → данные сразу у вас

Что сказать Cursor:
```
Сделай webhook-приёмник на Flask.
Когда приходит POST-запрос на /webhook —
сохраняй данные в Google Sheets и отправляй алерт в Telegram.
```

Cursor создаст сервер, который слушает входящие запросы и обрабатывает их.

Результат: Ты понимаешь как работают webhooks и можешь их использовать.

## Шаг 8. MCP — подключение сервисов к Cursor

MCP (Model Context Protocol) — способ подключить внешние сервисы прямо в Cursor. После подключения AI может сам обращаться к этим сервисам.

Например: подключаешь GitHub MCP → Cursor может создавать issues, PR, читать код из репозитория.

### Вариант 1: через настройки Cursor
1. Открой Cursor Settings (шестерёнка)
2. Перейди в раздел MCP
3. Нажми Add MCP Server
4. Выбери сервер из списка или вставь команду
5. Перезагрузи Cursor

### Вариант 2: через чат Cursor (самый простой)
Просто напиши в контекстное окно:
```
Подключи MCP-сервер для GitHub.
Мой токен: ghp_xxx
```
Cursor сам создаст нужный файл конфигурации.

### Вариант 3: файл конфигурации
Создай `.cursor/mcp.json` в проекте:
```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_..."
      }
    }
  }
}
```

Перезапусти Cursor: Ctrl+Shift+P → Reload Window. В Agent Mode теперь доступны инструменты GitHub.

> **Заметка (2026):** GitHub выпустили официальный сервер `github/github-mcp-server` (Docker / remote endpoint). Он надёжнее и активно обновляется. Пакет `@modelcontextprotocol/server-github` ещё работает и подходит для обучения, но в проде смотри в сторону официального — см. `https://github.com/github/github-mcp-server`.

Подробные примеры — в `04_дополнительные_материалы/mcp_config.md`, а также в слайдах `04_дополнительные_материалы/Презентации/MCP_подключение.html` и `04_дополнительные_материалы/Презентации/гайд_MCP.html`.

Результат: Ты понимаешь что такое MCP и как подключить внешний сервис к Cursor.

## Шаг 9. RAG и контекст — в чём разница

**Контекст** (то, что мы делали в модуле 02): файлы лежат в папке проекта, Cursor читает их напрямую. AI видит всё, что в папке. Подходит для 10-50 файлов. Не требует настройки.

**RAG** (Retrieval Augmented Generation): документы индексируются в базу, AI сначала ищет нужный фрагмент, потом отвечает. Подходит для 1000+ документов. Нужна настройка (embeddings, база).

На практике: для большинства задач хватает контекста — просто положите файлы в папку. RAG нужен, когда данных слишком много.

Результат: Ты понимаешь когда нужен RAG, а когда достаточно контекста.

## Шаг 11. Безопасность ключей

Правила, которые нельзя нарушать:

1. **Ключи только в .env**, никогда в коде
2. **.env всегда в .gitignore**
3. Если ключ попал на GitHub — немедленно отзови и создай новый
4. Каждый сервис → свой ключ
5. Ограничивай права: если ключу нужен только read — не давай write

Как проверить что .env не попал на GitHub:
```bash
git log --all --diff-filter=A -- .env
```

Если показывает результаты — .env когда-то добавлялся в репозиторий. Нужно очистить историю (например, через `git filter-repo`) и обязательно создать новые ключи — старые считай скомпрометированными.

Результат: Ты понимаешь правила безопасности API-ключей.
