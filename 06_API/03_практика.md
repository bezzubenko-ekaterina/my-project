# Практика — Модуль 06. API и интеграции

## Задание 1. Telegram-уведомление (повтори по инструкции)

1. Создай бота в @BotFather или используй существующего из модуля 05
2. Узнай свой CHAT_ID через @userinfobot
3. Сохрани токен и CHAT_ID в `.env`
4. Попроси Cursor написать скрипт, который отправляет сообщение в Telegram
5. Запусти — проверь что сообщение пришло

**Готово, когда:** Сообщение пришло в Telegram от вашего бота.

## Задание 2. Уведомление с датой (сделай сам)

Измени скрипт из задания 1, чтобы он отправлял текущую дату и время.

Попроси Cursor сам — не копируй промпт из инструкции. Опиши своими словами:
- Что должен делать скрипт
- Какой формат даты/времени
- Что именно писать в сообщении

**Готово, когда:** Сообщение с текущей датой и временем пришло в Telegram.

## Задание 4. Связка Telegram + Google Sheets (сделай сам)

Создай бота, который:
1. Принимает сообщения от клиентов
2. Записывает данные в Google Sheets
3. Отправляет уведомление менеджеру в Telegram

Попроси Cursor собрать эту связку. Опиши что должно происходить.

**Готово, когда:** Бот работает — клиент пишет, данные в таблице, уведомление менеджеру.

## Задание 5. MCP-сервер в Cursor (творческое)

1. Подключи один MCP-сервер в Cursor (варианты: GitHub, Filesystem, Notion, Playwright)
2. Используй любой из трёх способов: через настройки, через чат Cursor или через файл `.cursor/mcp.json`
3. Попроси Cursor сделать что-то через этот MCP-сервер

Примеры задач:
- **GitHub MCP** — «Покажи мои открытые issues»
- **Filesystem MCP** — «Прочитай файлы из папки Documents»
- **Notion MCP** — «Создай новую страницу в моём workspace»

**Готово, когда:** MCP-сервер подключён, Cursor успешно выполнил задачу через него.
